## Step 1: Plan Your Fields

### 1.1 Identify Your Needs
- **Event-Specific Information**
  - Session preferences
  - Dietary restrictions
  - T-shirt size
  - Skill level

- **Business Intelligence**
  - How did you hear about us?
  - Job title/company
  - Areas of interest
  - Purchase intent

### 1.2 Field Requirements
1. **Required vs. Optional**
   - Mark critical fields as required
   - Keep optional fields to a minimum

2. **Field Types** (See [Field Types](#field-types))
   - Text
   - Dropdown
   - Checkbox
   - Date picker
   - File upload

## Step 2: Create Custom Fields

### 2.1 Add a New Field
1. Go to **Event Dashboard** > **Registration** > **Custom Fields**
2. Click **+ Add Field**
3. Select field type
4. Enter field name and description
5. Configure settings (see Step 3)
6. Click **Save**

### 2.2 Field Types

| Type | Best For | Example |
|------|----------|---------|
| **Text** | Short answers | Company name |
| **Text Area** | Long responses | Feedback |
| **Dropdown** | Single selection | T-shirt size |
| **Multiple Choice** | Multiple selections | Session interests |
| **Checkbox** | Yes/No options | Dietary needs |
| **Date** | Date selection | Arrival date |
| **File Upload** | Documents, photos | ID verification |
| **Number** | Numeric input | Years of experience |

## Step 3: Configure Field Settings

### 3.1 Basic Settings
- **Field Label**: Clear, concise name
- **Help Text**: Instructions or examples
- **Required/Optional**: Mark if mandatory
- **Visibility**: Show/hide based on ticket type

### 3.2 Advanced Options
- **Default Value**: Pre-filled response
- **Validation Rules**: Format requirements
- **Conditional Logic**: Show/hide based on other fields
- **Character Limit**: For text fields
- **Options List**: For dropdowns/multiple choice

## Step 4: Add to Registration Forms

### 4.1 Form Placement
1. Go to **Registration** > **Form Builder**
2. Drag & drop custom fields
3. Organize into logical sections
4. Add section headers if needed

### 4.2 Conditional Fields
1. Select a field
2. Click **Add Conditional Logic**
3. Set conditions (e.g., "Show if 'Attending Workshop' is Yes")
4. Choose which fields to show/hide

## Step 5: Manage Check-in Data

### 5.1 Collecting Data
- **Pre-Event**: Via registration
- **Onsite**: During check-in
- **Post-Event**: Through follow-up surveys

### 5.2 Data Validation
- Required field indicators
- Format validation
- Real-time error messages

## Step 6: Export & Analyze

### 6.1 Exporting Data
1. Go to **Reports** > **Attendee Data**
2. Select fields to include
3. Choose format (CSV/Excel/PDF)
4. Click **Export**

### 6.2 Analyzing Responses
- **Segmentation**: Filter by field values
- **Trends**: Track changes over time
- **Correlations**: Find relationships between fields
- **Visualization**: Create charts and graphs

## Pro Tips & Best Practices

### Field Design
1. **Keep it Short**
   - Use clear, concise labels
   - Limit number of fields
   - Group related fields

2. **Smart Defaults**
   - Pre-fill known information
   - Use conditional logic
   - Set intelligent defaults

### Data Quality
1. **Validation Rules**
   - Required fields
   - Format requirements
   - Character limits

2. **User Experience**
   - Logical field order
   - Mobile-friendly design
   - Clear instructions

## Troubleshooting

### Common Issues & Solutions

#### "Field Not Showing"
1. Check field visibility settings
2. Verify conditional logic
3. Clear browser cache

#### "Data Not Saving"
1. Check required fields
2. Verify field validation
3. Test with different browser

#### "Export Issues"
1. Check field permissions
2. Try different export format
3. Export smaller data sets
